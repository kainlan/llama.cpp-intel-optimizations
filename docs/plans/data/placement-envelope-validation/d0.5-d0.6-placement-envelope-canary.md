# D0.5/D0.6 Placement Envelope Runtime Canary

**Date:** 2026-04-25
**Epic:** `llama.cpp-3h5gm` Unified Memory Placement
**Canary:** `tests/test-placement-envelope-canary.cpp`
**Status:** D0.5 shape PASS, D0.5 admission FAIL, D0.6 lifecycle FAIL

## What Was Added

`test-placement-envelope-canary` is a dedicated runtime probe for the remaining
placement-envelope claims:

- load one model with `llama_model_params.n_ctx_hint` and `n_ubatch_hint`,
- create/destroy in-envelope contexts with split KV and unified KV,
- create contexts concurrently against the same loaded model,
- optionally assert that over-envelope context creation is rejected.

It is buildable but not registered in default `ctest`, because the current
branch is expected to fail the over-envelope assertion until A6 is implemented.

Build:

```bash
./scripts/sycl-build.sh -r test-placement-envelope-canary
```

## D0.6 Lifecycle Finding

The same runs show monotonic host pinned/SCRATCH growth while contexts are
created and destroyed. Representative log lines:

```text
[SYCL] Pinned pool grow_zone: zone 3 grown by 2048.0 MB (1 new chunks, zone now 2490.2 MB)
[HOST-ARENA] SCRATCH zone grown by 33.9 MB for compute buffers
[SYCL] Pinned pool grow_zone: zone 3 grown by 2048.0 MB (1 new chunks, zone now 4538.2 MB)
[HOST-ARENA] SCRATCH zone grown by 35.4 MB for compute buffers
[SYCL] Pinned pool grow_zone: zone 3 grown by 2048.0 MB (1 new chunks, zone now 6586.2 MB)
[HOST-ARENA] SCRATCH zone grown by 33.4 MB for compute buffers
```

In the longer run, the pinned pool reached `total=32.0 GB` after repeated
context creation for compute requests of only ~30-36 MiB. This fails D0.6's
acceptance requirement that repeated context/slot plan creation/destruction
return arena allocations to unified_cache without monotonic growth.

## Plan Impact

- D0.5 remains open until over-envelope requests fail deterministically before
  allocation.
- D0.6 remains open and now has a concrete lifecycle failure: context compute
  buffers grow host SCRATCH/pinned chunks monotonically.
- A6 should validate requested shape against the loaded envelope.
- D11 / runtime arena work must implement the two-layer fix:
  - capacity layer: replace additive `grow_host_scratch_zone(total_bytes)` with
    reserve/admission semantics for the loaded placement envelope.
  - lifetime layer: split context/slot-lifetime host compute buffers into a
    `HOST_COMPUTE` / `HOST_RUNTIME` class that is returned on context/slot
    teardown; keep graph scratch reset-scoped separately.
- `llama.cpp-3h5gm.1` tracks the concrete SCRATCH/pinned lifecycle fix.
