constructions AND across pre- vs post-forward-pass on the same context.
Uses `llama_state_get_size(ctx)` as the public-API proxy for
per-backend scheduler reserve sizes (the true per-backend size is not
exposed on `llama_context` today). Runs on Mistral 7B Q4_0 (both models
if GPT-OSS 20B is available) with `n_ctx=4096`, `n_batch=512`,
`n_ubatch=512`, `flash_attn_type=AUTO`. Method: sequential
create-destroy-recreate, with one ctx kept alive for the pre/post
forward comparison. A bonus probe guarded by
`D0_1_PROBE_MULTICONTEXT=1` exists to re-exercise the original
two-context wedge hypothesis once the backend is fixed.

`llama_init_from_model` → `ggml_sycl_preload_model_weights` →

`EXIT=124` at the outer timeout.

See also D0.4: the same L0 hang signature reproduces via a call path
that does NOT touch the staging pool, which proves the bug is broader
than the staging-pool amplifier pattern. See plan doc §Known issues
